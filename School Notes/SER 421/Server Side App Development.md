# Server Side Web Frameworks
## Web Frameworks
### Server Side Driven Applications
- Server driven applications: Makes requests to retrieve content or process input
- Server responds with HTML
- Advantages:
	- Less complex and more secure
	- Still used and useful for some low bandwidth and low client computational power situations
	- Page rendering doesn't jump around
- Disadvantages
	- Poorer responsivity, UX
	- Lacks sophisticated application functionality
	- Limits server scalability
### Hybrid Applications
- Server and client share responsibility for behaviors
- Often used in modern content management systems
	- Server loads a page
	- Client makes subsequent requests to get personalized content
- Advantages
	- Improved responsivity
	- Personalization
	- Page rendering starts quickly
- Disadvantages
	- Poorly designed apps do not maintain a clean SoC between the client and server
