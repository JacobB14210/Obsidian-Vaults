# Team Review Deliverables
- Assessed every two weeks
	- Does not need to align with project sprint start and end dates
- Three deliverable items
	- Design/development artifacts since last team review
	- Project plan
	- Individual contribution artifacts since last team review
- Best three of each of these will count towards grade
# Rubrics For Team Review Items
- Rubric for design and individual
- Individual Contribution Report
	- Do not include overhead work
	- Each column needs to be filled
- Design
	- Video of walkthrough of code
# Minimum Expectation for 100%
- Strong evidence of meaningful contribution
	- 6 or more days of contribution in the assessment period
	- No gaps of more than 3 days without contribution
	- Complete one or more tasks
	- Work represent 18 hours of effort
# Project Plan Details
- Current sprint
- Link to project
- Burndown chart
- Link to meeting minutes
- Sprint planning minutes
- Link to sprint retrospective minutes
- Summary of feedback from sponsor
- Description of implementations or features
- Description of major technical challenges solved
	- There can be no major technical challenges
# Project Planning
## Task Sizing
- Tasks can be done in one or two works days
	- 8 hours of work
- Number of tasks = number of days worked times number of team members times sprint length (weeks)
